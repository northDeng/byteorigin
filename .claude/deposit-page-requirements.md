# Deposit 页面改造需求

## 日期
2026-03-27

## 背景
当前 `/deposit` 页面是一个简单的表单页，需要改造成商品展示+详情+支付的完整流程。

## 需求概述

### 1. 列表页 `/deposit`
- 将小程序、H5开发、APP、PC网站开发、技术咨询改造为商品卡片样式
- 每行 3 个卡片，移动端 1 列
- 每个卡片有：配图/icon、服务名称、简短描述、定金起价、查看详情按钮
- 点击卡片进入详情页

### 2. 详情页 `/deposit/$serviceType`（新建）
- 每个服务类型对应一个详情页（通过路由参数区分）
- 页面包含：
  - 服务介绍
  - 可选配置项（用户可选择不同开发配置）
  - 联系信息填写（姓名、电话、需求描述）
  - 支付方式选择（微信支付 / 支付宝）
  - 支付按钮

### 3. 各服务配置项
| 服务 | 配置选项 |
|------|----------|
| 小程序 | 功能模块（用户系统/支付/地图/消息推送）、页面数量（5/10/20+）、是否含后台管理 |
| H5 | 类型（营销页/活动页/电商）、屏幕适配（仅手机/多端）、是否含动画效果 |
| App | 平台（iOS/Android/双端）、功能复杂度（基础/标准/复杂）、是否含后台 |
| PC网站 | 类型（企业官网/电商/后台系统）、页面数量、是否含CMS |
| 技术咨询 | 咨询类型（架构设计/技术选型/代码审查/团队培训）、时长（1h/半天/全天） |

### 4. 支付方式 - 微信H5支付
- 目标：能过微信支付审核，可唤起微信H5支付
- 前端实现：拿到后端返回的 `mweb_url` 后 `window.location.href = mweb_url` 跳转
- 页面上需提示用户「请用手机浏览器打开，不支持在微信内直接唤起」
- 后端逻辑暂不实现（`wechat.ts` 已有 TODO 占位）

### 5. 移动端适配
- 全页面支持移动端
- 列表页：sm以下1列，sm 2列，lg 3列
- 详情页：表单、配置项全部移动端友好

## 技术约束
- 使用 lucide-react 图标，不引入额外图片资源
- 遵循现有 cn() / LocalizedLink / shadcn/ui 组件规范
- 不修改后端逻辑，不修改 wechat.ts
- 路由：TanStack Router 文件路由，新建 `deposit.$serviceType.tsx`
